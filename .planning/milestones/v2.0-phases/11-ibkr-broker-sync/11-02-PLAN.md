- `ibkrGatewayUrl: string` — gateway URL input
- `ibkrSslVerify: boolean` — SSL verification toggle (default false — allow self-signed per research)
- `ibkrConnectionStatus: "unknown" | "connected" | "disconnected"` — green/red dot state
- `ibkrConnectionError: string | null` — error message from test
- `ibkrDiscoveredAccounts: Array<{ id: string; type: string }>` — from gateway discovery
- `ibkrAccountMappings: Array<{ ibkrAccountId: string; ledgerAccountId: string }>` — account links
- `ibkrLastSync: string | null` — ISO timestamp
- `ibkrLastSyncStatus: string | null` — "success" or "error: ..."
- `ibkrSyncing: boolean` — loading state for sync
- `ibkrSyncResults: Array<{ ibkrAccountId: string; ledgerAccountId: string; newCount: number; dupCount: number; errCount: number; error?: string }> | null` — results modal data
- `ibkrSyncDateRange: "7d" | "30d" | "custom"` — date range selector
- `ibkrSyncCustomStart: string` — custom start date
- `ibkrSyncCustomEnd: string` — custom end date

**3. Load broker settings on mount:** In the existing settings load useEffect, add reads for `ibkr_gateway_url`, `ibkr_ssl_verify`, `ibkr_account_mappings`, `ibkr_last_sync`, `ibkr_last_sync_status` from the settings response. Set state accordingly.

**4. Build the Broker tab UI** (rendered when `activeCategory === "broker"`):

**Section A: Gateway Connection**
- Text input for gateway URL with placeholder "https://localhost:5000"
- Checkbox for "Allow self-signed certificates" (toggling ibkrSslVerify, default checked/true meaning allow self-signed, which maps to `ibkr_ssl_verify = "false"` in settings — note the inversion: UI says "allow self-signed" = sslVerify false)
- "Test Connection" button that:
  1. Saves current URL and SSL settings via PUT /api/settings
  2. Calls GET /api/broker/ibkr/status
  3. Updates connection status (green dot = connected, red dot = disconnected)
  4. On success, also calls GET /api/broker/ibkr/accounts to discover sub-accounts and populates ibkrDiscoveredAccounts
- Green/red dot indicator next to the URL input showing connection status
- Error message text below if disconnected (e.g., "Gateway returned 401", "self signed certificate")

**Section B: Account Mapping**
- Only visible when ibkrDiscoveredAccounts has items (after successful Test Connection)
- Table with columns: IBKR Account ID, Type, Ledger Account (dropdown)
- Each discovered IBKR account gets a row
- Dropdown for "Ledger Account" populated from the AccountProvider's `accounts` list (use `useAccount()` hook)
- "[+ Link Another Account]" is not needed here since we show ALL discovered accounts — the user just selects which ones to map
- "Save Mappings" button that PUTs `ibkr_account_mappings` as JSON string to /api/settings
- "Unlink" button per mapping row that removes the mapping (sets dropdown to empty)

**Section C: Trade Sync**
- Only visible when at least one account is mapped
- Date range selector: three buttons/pills for "Last 7 Days", "Last 30 Days", "Custom"
- When "Custom" selected, show two date inputs (start, end)
- "Sync Now" button (primary style, blue) — disabled while syncing, shows spinner during sync
- On click: POST /api/broker/ibkr/sync with `{ dateRange, startDate?, endDate? }`
- After sync completes: show results in a modal (AlertModal pattern)

**Results Modal:**
- Title: "Sync Results"
- Grouped by account (ledger account name)
- Per account: list showing new trades (green check icon), duplicates/skipped (gray skip icon), errors (amber warning icon) with counts
- Footer showing total: "X new trades imported, Y duplicates skipped"
- Close button

**Section D: Sync Status**
- "Last synced: [relative time] ([absolute timestamp])" — shown when ibkrLastSync is set
- Status indicator: green "Success" or red "Error: ..." based on ibkrLastSyncStatus
- When no sync has occurred: "No sync performed yet"

**Styling:** Follow existing settings page patterns — dark-first Tailwind, consistent spacing, section headers with subtle borders. Use the same input/button styles already in use on other settings tabs.
