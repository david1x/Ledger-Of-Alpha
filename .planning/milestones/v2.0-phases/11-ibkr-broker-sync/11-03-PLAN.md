<task type="auto">
  <name>Task 1: IBKRPositionsWidget component</name>
  <files>components/dashboard/IBKRPositionsWidget.tsx</files>
  <action>
Create `components/dashboard/IBKRPositionsWidget.tsx` as a "use client" component.

**Props:** `{ size: "large" | "medium" | "compact" }` — follows the existing widget size pattern.

**State:**
- `positions: Array<{ symbol, quantity, unrealizedPnl, pnlPercent, direction, mktPrice, avgPrice, ibkrAccountId }>` — default `[]`
- `lastFetched: Date | null` — default `null`
- `loading: boolean` — default `false`
- `error: string | null` — default `null`
- `gatewayConfigured: boolean` — default `true` (optimistic, set to false if API returns "No gateway URL configured")

**Refresh callback:**
- Set loading true
- `fetch("/api/broker/ibkr/positions")`
- If `!res.ok`: check if error is "No gateway URL configured" -> set `gatewayConfigured = false`. Otherwise set error message, keep existing positions (stale data display).
- If success: set positions, set lastFetched to new Date(), clear error
- Set loading false

**Rendering by state:**

*When `!gatewayConfigured`:*
- Empty state: centered text "Connect to IBKR in Settings" with a subtle link/text pointing to settings. Use `Cable` icon from lucide-react.

*When error and positions exist (offline with stale data):*
- Yellow banner at top: `AlertTriangle` icon + "Gateway offline" + relative timestamp of lastFetched
- Show last known positions below the banner

*When error and no positions (offline, never fetched):*
- Red error state: `WifiOff` icon + error message + "Try refreshing" text

*When positions loaded (normal state):*

**Header area** (rendered inside the widget, not the card header — card header is handled by DashboardShell):
- `RefreshCw` icon button (spinning animation when loading via `animate-spin` class)
- "Last updated X ago" text (compute relative time: seconds/minutes ago). Show "Never" if lastFetched is null.

**Position table:**
- Columns vary by size:
  - `large`: Symbol, Direction, Qty, Avg Price, Mkt Price, Unrealized P&L, P&L %
  - `medium`: Symbol, Qty, Unrealized P&L, P&L %
  - `compact`: Symbol, Unrealized P&L
- Direction arrow: `▲` (green) for long, `▼` (red) for short
- Unrealized P&L: green text for positive, red for negative, formatted with $ sign and 2 decimals
- P&L %: formatted with sign and 2 decimals, same color coding
- Use `text-sm` for table text, `text-xs` for compact size

**Footer:**
- Total unrealized P&L: sum of all positions' unrealizedPnl, bold, same green/red coloring
- Label: "Total Unrealized P&L"

**Styling:** Dark-first Tailwind. Table uses `divide-y divide-slate-700/50`. No borders on the outer container (DashboardShell handles the card wrapper). Consistent with other widget components in `components/dashboard/`.

**Do NOT auto-fetch on mount.** Positions are on-demand only (user decision). The refresh button is the trigger. Show the "Never fetched" state initially.
