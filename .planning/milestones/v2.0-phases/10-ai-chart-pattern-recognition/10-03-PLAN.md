   - Props: `{ trades: Trade[] }`
   - Compute pattern stats from trades: filter to closed trades with ai_primary_pattern and non-null pnl. Reduce into a Record<string, { count, wins, totalPnl }> grouped by ai_primary_pattern. Map to sorted array (by count descending).
   - Render a table with columns: Pattern | Trades | Win Rate | Avg P&L

     - Pattern name (text-sm font-medium)
     - Trade count
     - Win rate as percentage with color: >= 60% emerald-400, >= 40% yellow-400, below red-400
     - Avg P&L formatted as currency with +/- coloring (emerald for positive, red for negative)
   - Clickable rows: on click, navigate to `/trades?ai_pattern=${encodeURIComponent(pattern)}` — but since there's no /trades page with that filter currently, instead use an expandable row pattern: clicking a row toggles visibility of a sub-list showing the individual trades (symbol, date, direction, P&L). Use a ChevronDown/ChevronUp toggle.
   - If no analyzed trades exist: show an empty state message "No AI-analyzed trades yet. Upload and analyze chart screenshots to see pattern performance."
   - Header: "Pattern Performance" with BrainCircuit icon
