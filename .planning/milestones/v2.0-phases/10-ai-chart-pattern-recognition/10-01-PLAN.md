   - Check hasMigration(db, "019_ai_analysis")
   - Get cols via pragma table_info(trades)

4. In `lib/types.ts`, add to the Trade interface (after checklist_items, before created_at):
   ```typescript
   ai_patterns?: string | null;        // JSON: AnalysisResult
   ai_screenshots?: string | null;     // JSON: string[] filenames
   ai_qa_history?: string | null;      // JSON: QAEntry[]
   ai_primary_pattern?: string | null; // denormalized pattern name
   ```

5. Create `lib/ai-vision.ts` — server-side only OpenAI wrapper:
   - Export interfaces: PatternResult (name, confidence, description), AnalysisResult (patterns[], primary_pattern, summary), QAEntry (question, answer, ts, screenshotFilename)
   - Export PATTERN_NAMES array with all 15 pattern types from CONTEXT.md plus "None Detected"
   - Export ANALYSIS_SCHEMA for json_schema response_format with strict:true
   - Export async function `analyzeChartScreenshot(apiKey: string, filename: string): Promise<AnalysisResult>`:
     - Read file from data/screenshots/ using fs/promises
     - Base64 encode, determine MIME from extension
     - Call GPT-4o with the chart analysis prompt from RESEARCH.md
     - Use response_format json_schema with strict:true
     - Use detail:"high" for the image
     - max_tokens: 800
     - Parse and return AnalysisResult
   - Export async function `askFollowUp(apiKey: string, filename: string, question: string, priorAnalysis: AnalysisResult): Promise<string>`:
     - Same base64 pattern, include prior analysis as assistant context
     - Return text response (not structured JSON)
     - max_tokens: 500

6. Create `app/api/screenshots/route.ts` — POST endpoint:
   - Auth via getSessionUser (return 401 if no user)
   - Guest check: return 403 for guests
   - Accept multipart formData, get "file" field
   - Validate: only image/png, image/jpeg, image/webp allowed; max 5MB
   - Generate filename: `${user.id}-${crypto.randomUUID()}.${ext}`
   - mkdir data/screenshots/ (recursive), writeFile
   - Return { filename, url: `/api/screenshots/${filename}` }

7. Create `app/api/screenshots/[filename]/route.ts` — GET and DELETE:
   - GET: Auth check, verify filename starts with `user.id + "-"` (security), readFile, return Response with correct Content-Type and Cache-Control: private, max-age=86400. Handle path traversal: reject filenames with "/" or "..".
   - DELETE: Auth check, verify ownership, unlink file with fs/promises, return { success: true }. Handle file-not-found gracefully.
