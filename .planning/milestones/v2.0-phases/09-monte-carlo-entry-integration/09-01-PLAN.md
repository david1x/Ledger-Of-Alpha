**public/mc-worker.js:**
Create a plain JS Web Worker file (must be in public/ for Next.js static serving). Copy the `runMonteCarlo` function logic from `lib/simulation.ts` directly into the worker (workers cannot import from TS modules). The worker listens for messages with shape `{ returns, startingBalance, numTrades, iterations, ruinThreshold }` and posts back `SimulationResult`. Also handle a second message type for binary search: `{ type: 'findSuggestedSize', returns, startingBalance, numTrades, iterations, maxRuinProb, entry, stopLoss, direction, commission }` -- this iterates position sizes (binary search between 1 and maxShares where maxShares = Math.floor(startingBalance / entry)) and finds the largest size where ruin probability stays under `maxRuinProb`. Posts back `{ type: 'suggestedSize', shares: number }`.

**lib/use-monte-carlo.ts:**
Create a `"use client"` React hook `useMonteCarloPreview` with this signature:

```typescript
interface MonteCarloInput {
  historicalTrades: { pnl_percent: number; strategy_id?: string | null }[];
  startingBalance: number;
  entry: number | null;
  stopLoss: number | null;
  direction: "long" | "short";
  currentShares: number | null;
  ruinThreshold: number;       // user-configurable, default 5 (percentage)
  strategyFilter: string | null; // null = "All Strategies"
  commission: number;
}

interface MonteCarloOutput {
  result: SimulationResult | null;
  suggestedShares: number | null;
  isRunning: boolean;
  tradeCount: number;          // how many trades used in simulation
  hasEnoughData: boolean;      // >= 10 trades
}
```

The hook should:
1. Create a Worker instance on mount (`new Worker('/mc-worker.js')`), terminate on unmount
2. Filter `historicalTrades` by `strategyFilter` if set (match on `strategy_id`)
3. Compute `returns` array as `trade.pnl_percent / 100` from filtered trades
4. Debounce 500ms before posting to worker (use a setTimeout ref, clear on input change)
5. Use `ruinThreshold / 100` as the ruin threshold parameter (convert from percentage)
6. Use 1K iterations (not 5K) per architecture decision
7. Use 100 numTrades (project forward 100 trades)
8. After receiving simulation result, also post `findSuggestedSize` message for binary search
9. Return `MonteCarloOutput` with computed values

**Layout (collapsible panel):**
- Section header with Activity icon (lucide-react) and title "Monte Carlo Risk Preview"
- Collapse/expand toggle (ChevronDown/ChevronUp). Default state from `defaultCollapsed` prop.
- When collapsed: show only the header row with a mini risk badge on the right side:
  - Green badge: "Low risk" when ruin < threshold
  - Yellow badge: "Moderate risk" when ruin >= threshold and < 2x threshold
  - Red badge with warning icon: "High risk: X.X%" when ruin >= 2x threshold
  - Gray badge: "No data" when insufficient inputs
- Badge updates in real-time even when collapsed.

**Expanded panel contents (top to bottom):**

1. **Risk warning banner** (conditional): If ruin probability exceeds threshold, show inline banner:
   - Yellow bg for moderate (1-2x threshold): "Ruin probability: X.X% (threshold: Y%)"
   - Red bg for high (>2x threshold): "Ruin: X.X% (threshold: Y%) -- Consider reducing position size"

2. **Low data warning** (conditional): When `tradeCount < 10`, show amber banner: "Not enough trade history (X/10 trades). Results may be unreliable."

3. **Results grid** (2x2 grid):
   - Median Outcome: formatted as currency with green/red color based on profit/loss
   - Prob. of Profit: percentage with green/red coloring
   - Max Drawdown: percentage in red tones
   - Ruin Probability: percentage with severity color (green/yellow/red)

4. **Position size suggestion row:**
   - Left: "MC Suggested: X shares"
   - Right: "Your current: Y shares" with delta note (e.g., "MC suggests 50% less risk" or "25% more room")
   - "Apply" button (emerald, small) that calls `onApplyShares(suggestedShares)`
   - Hide this row if entry/stopLoss not set or suggestedShares is null

5. **Strategy filter dropdown:**
   - Default: "All Strategies"
   - List strategies with trade count: "Wyckoff Buying (23 trades)"
   - Strategies with fewer than 5 matching trades are disabled with "(min 5)" note
   - Changing selection re-triggers simulation via the hook

6. **Loading state:** When `isRunning`, show a subtle pulse animation on the results grid.

**Styling:** Match existing dark-first Tailwind patterns. Use the same INPUT/LABEL style constants from TradeModal for the dropdown. Use slate-800/900 backgrounds, emerald accents, rounded-xl cards.

Use the `useMonteCarloPreview` hook internally to manage simulation state. The component owns its own `strategyFilter` state.
