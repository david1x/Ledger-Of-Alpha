  <action>
Create `components/dashboard/TemplatePanel.tsx` as a "use client" component that renders a dropdown panel triggered by a button.

Props interface:
```typescript
interface TemplatePanelProps {
  templates: (LayoutTemplate | BuiltInTemplate)[];
  onSave: (name: string) => void;
  onLoad: (template: LayoutTemplate | BuiltInTemplate) => void;
  onDelete: (templateId: string) => void;
  onSaveAs: (preset: BuiltInTemplate, newName: string) => void;
  isGuest: boolean;
}
```

Import the `LayoutTemplate` and `BuiltInTemplate` types from DashboardShell (they should be exported). If they are not exported, export them from DashboardShell in this task.

Component behavior:
1. **Trigger button**: A button with `BookOpen` icon (from lucide-react) that fits the existing toolbar pill style (`h-7 w-7 sm:h-8 sm:w-8 rounded-xl`). Title: "Templates".
2. **Dropdown panel**: Absolute-positioned below the trigger button. Uses `z-50`. Styled consistently with the export dropdown pattern already in DashboardShell (~line 982): `rounded-2xl border dark:border-slate-700 border-slate-200 dark:bg-slate-900 bg-white shadow-2xl`.
3. **Template list section**:
   - Each template row shows: name, and action buttons.
   - Built-in templates (check `'readonly' in template`): Show "Apply" button and "Copy" button (Copy icon). No delete button. No rename.
   - User templates: Show "Apply" button and "Delete" button (Trash2 icon).
4. **Apply confirmation**: Clicking "Apply" sets an `applyConfirm` state to the template id. The row changes to show "Apply this layout?" text with a green confirm button and a gray cancel button. Clicking confirm calls `onLoad(template)` and closes the dropdown. Clicking cancel clears the confirm state. This is the inline two-step confirmation per research recommendation.
5. **Save current section**: At the bottom of the dropdown, a row with a text input (placeholder "Template name...") and a Save button (BookmarkPlus icon). Disabled if input is empty or `isGuest`. Pressing Enter or clicking Save calls `onSave(name)`, clears the input, and shows a brief "Saved!" feedback (via local state, 1.5s timeout).
6. **Save as copy flow**: Clicking "Copy" on a built-in preset populates the save input with `"{preset.name} (copy)"` and focuses it. User can edit the name and click Save. Alternatively, directly call `onSaveAs(preset, inputName)` when saving with a copy source set.
7. **Close on outside click**: Use a `useRef` for the container and a `useEffect` with `mousedown` listener to close when clicking outside. Also close on Escape key.
8. **Guest mode**: If `isGuest`, show the template list (they can browse) but disable Save input and Delete buttons. Apply still works (changes layout in memory, just won't persist — that's handled by saveLayout's guest check).

Styling notes:
- Dark mode first per project convention
- Max height on template list with overflow-y-auto for scrollable list (unlikely to be many, but safe)
- Subtle divider between template list and save section
- Template names truncated with `truncate` class if long
- Width: `w-64` or `w-72` — enough for template names
