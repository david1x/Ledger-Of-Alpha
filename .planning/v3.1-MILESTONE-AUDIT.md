## Cross-Phase Integration

**Score:** 11/13 requirement integrations fully clean; 2 with minor wiring debt

### Issue 1: `finishEdit` Dead Code

**Severity:** Minor
**Affected:** PERSIST-01, RESIZE-01
**Location:** `components/dashboard/DashboardShell.tsx` line 1221

The edit mode toggle button calls `setEditMode(!editMode)` directly. The `finishEdit()` function (line 947) — which runs auto-compact and immediate save — is never called. Layout changes during edit mode are still saved via the 1s debounce timer, but:
- Auto-compact (filling grid gaps after reorder) does not fire on edit exit
- If user closes browser within 1s of last interaction, final state may not persist

**Fix:** Wire button to call `finishEdit()` when exiting edit mode.

### Issue 2: `resetLayout` Missing `_gridScale`

**Severity:** Cosmetic
**Affected:** RESIZE-04
**Location:** `components/dashboard/DashboardShell.tsx` line 955-959

`resetLayout` constructs `defaultLayout` without `_gridScale`. `saveLayout` stamps it before the API write, so no data loss occurs. However, the intermediate object is unguarded — same pattern that caused a migration loop bug earlier in Phase 27.

**Fix:** Add `_gridScale: GRID_COLS` to the `defaultLayout` object.
