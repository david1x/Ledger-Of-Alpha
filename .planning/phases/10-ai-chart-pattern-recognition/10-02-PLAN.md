```typescript
export interface PatternResult {
  name: string;           // e.g. "Bull Flag"
  confidence: number;     // 0.0 - 1.0
  description: string;    // 1-2 sentence explanation
}

export interface AnalysisResult {
  patterns: PatternResult[];   // top 2-3, sorted by confidence desc
  primary_pattern: string;     // patterns[0].name
  summary: string;             // overall chart context sentence
}

export interface QAEntry {
  question: string;
  answer: string;
  ts: string;
  screenshotFilename: string;
}
```

API routes (from Plan 01):
- POST /api/screenshots — FormData with "file" field → { filename, url }
- GET /api/screenshots/[filename] — serves image
- DELETE /api/screenshots/[filename] — deletes image
- POST /api/ai/analyze — { filename } → AnalysisResult
- POST /api/ai/followup — { filename, question, priorAnalysis } → { answer }

  <action>
Create three "use client" components in `components/ai/`:

**ScreenshotUploader.tsx** — The main container orchestrating the upload-analyze-link flow:
- Props: `{ trades: Trade[], onTradeLinked?: (tradeId: number, analysis: AnalysisResult, screenshots: string[], qaHistory: QAEntry[]) => void }`
- State: pendingFile (File|null), preview (string URL), uploadedFilename (string|null), analysisResult (AnalysisResult|null), qaHistory (QAEntry[]), uploading (boolean), analyzing (boolean), linkedTradeId (number|null)
- **Upload zone**: A bordered dashed area (dark:border-slate-600) with drag-and-drop handlers (onDragOver, onDrop) and hidden file input triggered by click. Text: "Drop chart screenshot or click to upload". Accept: image/png, image/jpeg, image/webp.
- On file select/drop: validate type and 5MB size client-side. Show toast error if invalid. Set preview via URL.createObjectURL(). Show the file with thumbnail, filename, file size in KB/MB.
- **Preview state**: Show thumbnail (max-h-48 object-contain), filename, file size. Three buttons:
  - "Analyze" (emerald, BrainCircuit icon) — calls /api/screenshots POST then /api/ai/analyze POST
  - "Remove" (red text, X icon) — if uploaded, calls DELETE /api/screenshots/[filename]; clears all state
  - If already analyzed: show "Re-analyze" instead with a tooltip warning "Re-analyzing clears Q&A history"
- **Analyze flow**: Upload file first (POST /api/screenshots), then analyze (POST /api/ai/analyze with returned filename). Show spinner during both steps. On success: render PatternResults below. On NO_API_KEY error code: show message with link to Settings (/settings?tab=integrations).
- **Trade linking**: After analysis, show a dropdown to link to a trade. Use a simple select element populated from the trades prop (show symbol + date for each). On link: PATCH /api/trades/[id] with { ai_patterns: JSON.stringify(analysisResult), ai_screenshots: JSON.stringify([filename]), ai_qa_history: JSON.stringify(qaHistory), ai_primary_pattern: analysisResult.primary_pattern }. Call onTradeLinked callback.
- **Pattern history hint**: After analysis completes, fetch trades with the same ai_primary_pattern and display inline: "You've traded [pattern] X times (Y% win rate)" with a link to /analytics. Use a simple fetch to /api/trades?ai_pattern=X (will be handled in Plan 03, but prepare the UI).
- Dark-first Tailwind styling consistent with the rest of the app.

**PatternResults.tsx** — Displays analysis results:
- Props: `{ result: AnalysisResult }`
- Render the summary as a subtle italic text at the top
- For each pattern in result.patterns:
  - Pattern name (bold, text-sm)
  - Confidence bar: a `<div>` with percentage width inside a slate-700 background bar. Color: >= 0.7 emerald-500, >= 0.4 yellow-500, below amber-600.
  - Confidence percentage text (e.g. "85%")
  - Description text (text-xs text-slate-400)
- Highlight the primary pattern with a subtle left border (border-l-2 border-emerald-500)

**FollowUpChat.tsx** — Follow-up Q&A interface:
- Props: `{ filename: string, priorAnalysis: AnalysisResult, qaHistory: QAEntry[], onQAUpdate: (history: QAEntry[]) => void }`
- **Quick chips**: Small pill-shaped buttons for "Supply/Demand Zones", "Support/Resistance Levels", "Trend Direction". Style: rounded-full px-3 py-1 text-xs bg-slate-700 hover:bg-slate-600 transition. On click: send that text as the question.
- **Freeform input**: A text input with send button (Send/ArrowUp icon). Placeholder: "Ask about this chart..."
- **Q&A history**: Render each QAEntry as:
  - Question in a right-aligned bubble (bg-emerald-900/30, rounded)
  - Answer in a left-aligned block (text-slate-300, with a small BrainCircuit icon)
- On send: POST /api/ai/followup with { filename, question, priorAnalysis }. Add loading state. On response: append new QAEntry to qaHistory (with ts: new Date().toISOString(), screenshotFilename: filename). Call onQAUpdate with updated history.
