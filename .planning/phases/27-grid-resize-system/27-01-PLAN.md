**Step A: Create `components/dashboard/useGridResize.ts`**

Create a custom hook `useGridResize` that handles SE corner drag-to-resize with snap-to-grid behavior.

```typescript
export interface WidgetDims { w: number; h: number }

interface UseGridResizeOptions {
  gridRef: React.RefObject<HTMLDivElement | null>;  // ref to the grid container
  cols: number;          // total grid columns (6)
  gap: number;           // gap in px (12 = gap-3)
  onResize: (id: string, dims: WidgetDims) => void;
}

interface ResizeState {
  id: string;
  startX: number;
  startY: number;
  startW: number;
  startH: number;
  colPx: number;   // computed pixel width of one column
  rowPx: number;   // computed pixel height of one row (use first visible card's offsetHeight or a default of 200)
}

export function useGridResize({ gridRef, cols, gap, onResize }: UseGridResizeOptions) {
  // Returns: { resizingId, onResizeStart }
  // resizingId: string | null — which widget is currently being resized (for visual feedback)
  // onResizeStart: (e: React.PointerEvent, id: string, currentW: number, currentH: number) => void
}
```

Implementation details:
- On pointerdown on the resize handle: compute `colPx` from gridRef's clientWidth (`(gridWidth - (cols - 1) * gap) / cols`). Compute `rowPx` from the card element's own offsetHeight divided by its current `h` value (or use a default of 200px if h=1). Store start position and dims in a ref.
- Attach pointermove and pointerup listeners to `document` (not the handle — handle is tiny).
- On pointermove: compute deltaX from startX. New w = `startW + Math.round(deltaX / (colPx + gap))`. Clamp w to [1, 6]. Only call `onResize` if w changed from current value.
- `e.stopPropagation()` on pointerdown to prevent @dnd-kit from interpreting resize as drag. The PointerSensor has `activationConstraint: { distance: 5 }` — the resize handle's stopPropagation prevents the drag sensor from even seeing the event.
- Set `pointer-events: none` on an overlay div during resize to prevent iframes/charts from stealing events (same pattern as PersistentChart.tsx sidebar resize).
- Clean up listeners on pointerup and on unmount.
- For now, only handle `w` (column) dimension. Row (`h`) will be added in Plan 02.
