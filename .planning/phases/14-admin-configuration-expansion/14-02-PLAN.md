  <action>
All three endpoints follow the same pattern: POST handler, `requireAdmin` guard, read key/config from `_system` settings in DB, attempt validation, return `{ ok: true }` or `{ ok: false, error: string }`.

1. `app/api/admin/test-smtp/route.ts`:
   - Import `nodemailer`, `requireAdmin`, `getDb`, `NextRequest`, `NextResponse`
   - Read smtp_host, smtp_port, smtp_secure, smtp_user, smtp_pass from `_system` settings
   - If host/user/pass are missing, return `{ ok: false, error: "SMTP not configured" }`
   - Create nodemailer transport with those settings, call `transport.verify()`
   - On success return `{ ok: true }`, on error return `{ ok: false, error: err.message }`
   - Wrap the verify call with a try/catch

2. `app/api/admin/test-fmp/route.ts`:
   - Import `requireAdmin`, `getDb`, `NextRequest`, `NextResponse`
   - Read `fmp_api_key` from `_system` settings
   - If empty, return `{ ok: false, error: "FMP API key not configured" }`
   - Fetch `https://financialmodelingprep.com/stable/search-symbol?query=AAPL&apikey=${apiKey}` with `AbortSignal.timeout(5000)`
   - Parse JSON response. FMP returns `{ "Error Message": "..." }` on invalid key
   - If `!res.ok` or `data["Error Message"]`, return `{ ok: false, error: data["Error Message"] ?? "API error" }`
   - Otherwise return `{ ok: true }`

3. `app/api/admin/test-gemini/route.ts`:
   - Import `GoogleGenerativeAI` from `@google/generative-ai`, `requireAdmin`, `getDb`, `NextRequest`, `NextResponse`
   - Read `openai_api_key` from `_system` settings (yes, the key name is `openai_api_key` for backward compat — the service is Gemini)
   - If empty, return `{ ok: false, error: "Gemini API key not configured" }`
   - Create `new GoogleGenerativeAI(apiKey)`, get model `"gemini-2.5-flash"`, call `model.generateContent("Say OK")`
   - On success return `{ ok: true }`, on error return `{ ok: false, error: err.message }`
   - This consumes a tiny amount of API quota — acceptable for manual admin test
