Create `components/trades/SummaryStatsBar.tsx` as a "use client" component with the following structure:

**Props interface:**
```typescript
interface SummaryStatsBarProps {
  filteredTrades: Trade[];
  allTrades: Trade[];
  accountSize: number;
}
```

**Stats computation (single useMemo):**
- Filter to `status === "closed"` trades only (exclude open trades)
- Cumulative Return: sum of pnl for all closed filtered trades
- P/L Ratio: average winning trade pnl / |average losing trade pnl|. Handle division by zero: if no losers, set to Infinity; if no winners and no losers, set to 0
- Win %: (winners count / closed count) * 100

**Sparkline data (second useMemo):**
- Sort closed filtered trades by exit_date ascending
- Cumulative Return sparkline: running sum of pnl (one point per trade)
- P/L Ratio sparkline: rolling 10-trade window P/L ratio
- Win % sparkline: rolling 10-trade window win rate
- For empty data, use `[{ value: 0 }, { value: 0 }]` as fallback to prevent layout collapse

**Card layout:**
- 3-column CSS grid: `grid grid-cols-3 gap-4`
- Each card: `dark:bg-slate-800 bg-white rounded-xl shadow p-4 flex flex-col gap-1`
- Card structure top-to-bottom: label (uppercase xs text), headline value (text-2xl font-bold, color-coded), subtitle (xs slate text), sparkline (h-12 container at bottom with mt-auto pt-2)

**Color coding per card:**
- Cumulative Return: emerald-400 if >= 0, red-400 if < 0
- P/L Ratio: emerald-400 if > 1.0, red-400 if <= 1.0
- Win %: emerald-400 if > 50, red-400 if <= 50

**Headline display formatting:**
- Cumulative Return: `+$1,234.56` or `-$1,234.56` (with sign prefix and comma formatting)
- Cumulative Return subtitle: `X.X% of account` (totalPnl / accountSize * 100)
- P/L Ratio: `X.XX` (toFixed(2)). If Infinity (no losers), display as dash "—"
- P/L Ratio subtitle: `Avg $X.XX / $Y.XX`. If no losers: "No losing trades". If no winners: "No winning trades"
- Win %: `XX.X%` (toFixed(1))
- Win % subtitle: `XW / YL` (winner count / loser count)

**Sparkline rendering (recharts):**
- Use `ResponsiveContainer` width="100%" height="100%" wrapping `AreaChart`
- Strip all interactivity: no XAxis, YAxis, CartesianGrid, Tooltip
- Each Area: `type="monotone"`, `dataKey="value"`, `dot={false}`, `activeDot={false}`, `isAnimationActive={false}`
- Each chart needs a UNIQUE gradient ID to avoid SVG collision: `statGradCumulative`, `statGradPL`, `statGradWin`
- Gradient colors: match the card's headline color (emerald hex #22c55e or red hex #ef4444) for Cumulative Return and Win %. For P/L Ratio, use neutral blue #3b82f6 (avoids confusing color flips as the rolling ratio crosses 1.0)
- Gradient: 35% opacity at top (offset 5%), 0% opacity at bottom (offset 95%)
- strokeWidth={1.5}

**Privacy masking:**
- Call `usePrivacy()` at component top level
- When `hidden` is true: replace all dollar amounts, percentages, counts, and ratio numbers with `"------"`
- Sparklines remain visible when privacy mode is active (shapes don't reveal actual values)
- The "Based on X of Y trades" label counts should also be masked

**"Based on X of Y trades" label:**
- Render below the 3-column grid only when `filteredTrades.length < allTrades.length` (i.e., filters are active)
- Styling: `text-center text-xs dark:text-slate-500 text-slate-400 mt-2`
- When hidden: show "Based on -- of -- trades"

**Edge cases:**
- 0 closed trades: show $0.00 / 0% / 0.0% with flat sparklines, cards remain at full size
- All winners (no losers): P/L ratio headline "—", subtitle "No losing trades"
- All losers (no winners): P/L ratio "0.00", subtitle "No winning trades"
- Single trade: sparkline shows one point (recharts handles gracefully with the 2-point fallback)
