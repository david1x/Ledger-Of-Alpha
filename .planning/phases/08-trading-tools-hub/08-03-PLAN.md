**Symbol Selection:**

**Time Period Selector:**
- Dropdown select with options: 1D, 5D, 30D, 60D, 90D, 180D, YTD, 365D (default: 90D)
- Map UI periods to API range values:
  ```
  "1D" -> "1d", "5D" -> "5d", "30D" -> "1mo", "60D" -> "3mo",
  "90D" -> "3mo", "180D" -> "1y", "YTD" -> "1y", "365D" -> "1y"
  ```
- For YTD: after fetching with range "1y", filter bars client-side to only include timestamps >= Jan 1 of current year

**Calculate Button:**
- Explicit "Calculate" button (NOT live update — per user decision since it requires external data)
- Disabled when < 2 symbols selected
- Shows loading state during fetch

**Sequential OHLCV Fetching (per architecture decision — NEVER Promise.all):**
- `for` loop with `await` for each symbol
- Call `GET /api/ohlcv?symbol=${sym}&interval=1d&range=${mappedRange}`
- Progress state: `{ current: number; total: number; symbol: string }`
- Render progress bar: filled portion = current/total, label: "Fetching {symbol} ({current}/{total})"
- Store results as `Record<string, { time: number; close: number }[]>`

**Series Alignment (critical — per research pitfall):**
- Before computing correlation for any pair (A, B), align by common timestamps
- Build a Map of timestamps for series B, filter series A to only include matching timestamps
- Both aligned arrays must have length >= 2, otherwise correlation = 0

**Correlation Computation:**
- Import `{ sampleCorrelation }` from "simple-statistics" (NAMED import, not default)
- Build NxN matrix: `symbols.map(a => symbols.map(b => ...))`
- Diagonal = 1.0 always
- For each off-diagonal pair: align series, call `sampleCorrelation(alignedA, alignedB)`

**Matrix Display:**
- NxN grid rendered as an HTML table
- Column and row headers = symbol names (rotated -45deg for columns if > 5 symbols, otherwise horizontal)
- Each cell shows correlation value to 2 decimal places
- Cell background color: interpolate from red (-1) through white/neutral (0) to green (+1)
  - Negative: red-500 with opacity proportional to absolute value
  - Positive: emerald-500 with opacity proportional to value
  - Zero: transparent/neutral
  - Use inline style: `backgroundColor: val > 0 ? rgba(16,185,129, val*0.6) : rgba(239,68,68, Math.abs(val)*0.6)`
- Diagonal cells (1.00) in a distinct muted style (e.g. dark:bg-slate-700)

**Empty/Error States:**
- No symbols selected: "Add at least 2 symbols to calculate correlations"
- Fetch error for a symbol: show warning icon next to that symbol's chip, skip it in matrix
- Insufficient data for a pair: show "N/A" in cell instead of 0

**Styling:** Dark-first Tailwind. Matrix should be scrollable horizontally if > 6 symbols.
