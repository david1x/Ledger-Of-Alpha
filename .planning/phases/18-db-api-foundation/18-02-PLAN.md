Create `app/api/trades/[id]/mistakes/route.ts` with POST and DELETE handlers.

POST handler (tag a trade with a mistake type):
- If `isGuest(req)` return 403 with "Guests cannot tag trades."
- `getSessionUser(req)`, return 401 if null.
- `const { id: tradeId } = await params;` (Next.js 15 awaited params pattern).
- Parse body: `{ mistake_id }`.
- Verify trade ownership: `SELECT id FROM trades WHERE id = ? AND user_id = ?` using `Number(tradeId)` and `user.id`. Return 404 if not found.
- Verify mistake type ownership: `SELECT id FROM mistake_types WHERE id = ? AND user_id = ?` using `mistake_id` and `user.id`. Return 404 if not found.
- Insert: `INSERT OR IGNORE INTO trade_mistake_tags (trade_id, mistake_id) VALUES (?, ?)` with `Number(tradeId)` and `mistake_id`.
- Return `{ tagged: true }` with status 201.
- Wrap in try/catch with error logging.

DELETE handler (untag a mistake from a trade):
- Guest check (403), auth check (401).
- `const { id: tradeId } = await params;`
- Parse body: `{ mistake_id }`.
- Verify trade ownership. Return 404 if not found.
- `DELETE FROM trade_mistake_tags WHERE trade_id = ? AND mistake_id = ?` with `Number(tradeId)` and `mistake_id`.
- Return `{ deleted: true }`.

Import pattern: `import { NextRequest, NextResponse } from "next/server"`, `import { getDb } from "@/lib/db"`, `import { getSessionUser, isGuest } from "@/lib/auth"`.

CRITICAL: Both POST and DELETE must verify BOTH trade ownership AND mistake type ownership (POST only — DELETE can skip mistake check since the row just gets deleted). This prevents cross-user data manipulation.
