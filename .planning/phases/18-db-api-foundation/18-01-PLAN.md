CREATE TABLE IF NOT EXISTS trade_mistake_tags (
  trade_id   INTEGER NOT NULL REFERENCES trades(id) ON DELETE CASCADE,
  mistake_id TEXT NOT NULL REFERENCES mistake_types(id) ON DELETE CASCADE,
  PRIMARY KEY (trade_id, mistake_id)
);
CREATE INDEX IF NOT EXISTS idx_trade_mistake_tags_trade ON trade_mistake_tags(trade_id);
CREATE INDEX IF NOT EXISTS idx_trade_mistake_tags_mistake ON trade_mistake_tags(mistake_id);

Create `app/api/mistakes/route.ts` with GET and POST handlers:

GET handler:
- If `isGuest(req)` return `NextResponse.json([])` (empty array, not 403).
- Otherwise `getSessionUser(req)`, return 401 if null.
- Query: `SELECT * FROM mistake_types WHERE user_id = ? ORDER BY created_at ASC`.
- Wrap in try/catch, log errors, return 500 on failure.

POST handler:
- If `isGuest(req)` return 403 with "Guests cannot create mistake types."
- `getSessionUser(req)`, return 401 if null.
- Parse body: `{ name, color }`. Validate `name?.trim()` is truthy, return 400 if empty.
- Generate `id` via `crypto.randomUUID()`.
- INSERT with parameterized query. Default color to `'#ef4444'` if not provided.
- On SQLITE_CONSTRAINT_UNIQUE error, return 409 with "A mistake type with this name already exists."
- Return the created row with status 201.

Create `app/api/mistakes/[id]/route.ts` with PUT and DELETE handlers:

PUT handler:
- Guest check (403), auth check (401).
- `const { id } = await params;` (Next.js 15 pattern — params is a Promise).
- Parse body: `{ name, color }`.
- Verify ownership: `SELECT id FROM mistake_types WHERE id = ? AND user_id = ?`. Return 404 if not found.
- If `name !== undefined`, UPDATE name (trimmed). If `color !== undefined`, UPDATE color.
- Handle SQLITE_CONSTRAINT_UNIQUE for duplicate name (409).
- Return the updated row.

DELETE handler:
- Guest check (403), auth check (401).
- `const { id } = await params;`
- Verify ownership. Return 404 if not found.
- `DELETE FROM mistake_types WHERE id = ?` — FK cascade handles trade_mistake_tags cleanup automatically.
- Return `{ deleted: true }`.

Follow the exact import pattern from existing routes: `import { NextRequest, NextResponse } from "next/server"`, `import { getDb } from "@/lib/db"`, `import { getSessionUser, isGuest } from "@/lib/auth"`.
