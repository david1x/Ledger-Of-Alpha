Output: A new `lib/request-url.ts` utility and updated email/notification helpers that derive the correct public URL from each incoming request.
</objective>

1. Create `lib/request-url.ts` with a single exported function `getBaseUrl(req: NextRequest): string` that implements the priority chain from EMAIL-02:
   - Priority 1: Admin DB override — query `settings` table for `user_id = '_system'` and `key = 'app_url'`. Use `require("./db")` pattern (same as existing `getAppUrl()` in email.ts). Strip trailing slash.
   - Priority 2: `x-forwarded-host` + `x-forwarded-proto` headers — split both on comma, take first value, trim. Default proto to `"https"` if `x-forwarded-proto` absent. Return `${proto}://${host}`.
   - Priority 3: `host` header — infer `http` for localhost/127.* addresses, `https` otherwise. Return `${proto}://${host}`.
   - Priority 4: `process.env.NEXT_PUBLIC_APP_URL` — read inside function body (NOT at module level). Strip trailing slash.
   - Priority 5: Hardcoded fallback `"http://localhost:3000"`.
   - Wrap DB access in try/catch (same pattern as existing `getAppUrl`).
   - Import type only: `import type { NextRequest } from "next/server";`

2. Modify `lib/email.ts`:
   - Add optional `baseUrl?: string` parameter as the LAST argument to `sendVerificationEmail`, `sendPasswordResetEmail`, and `sendAlertEmail`.
   - In each function body, use `const base = baseUrl ?? getAppUrl();` then replace the `getAppUrl()` call in URL construction with `base`.
   - Do NOT modify `sendOtpEmail` — it contains no URLs.
   - Do NOT remove the internal `getAppUrl()` function — it serves as the fallback for callers without request context.
   - Do NOT change `isSmtpConfigured`, `getSmtpConfig`, `send`, or `escHtml`.
