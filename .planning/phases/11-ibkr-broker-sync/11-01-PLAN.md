Gateway HTTP wrapper using undici Agent for SSL toggle. Contains:

- `ibkrFetch(gatewayUrl: string, path: string, sslVerify: boolean, init?: RequestInit): Promise<Response>` â€” prepends `/v1/api` to path, creates `new Agent({ connect: { rejectUnauthorized: sslVerify } })` from `undici` when sslVerify is false, passes as `dispatcher`. When sslVerify is true, use default fetch (no custom dispatcher).

- `getIbkrSettings(db: Database.Database, userId: string): { gatewayUrl: string; sslVerify: boolean; accountMappings: Array<{ ibkrAccountId: string; ledgerAccountId: string }> }` â€” reads `ibkr_gateway_url`, `ibkr_ssl_verify`, `ibkr_account_mappings` from settings table for the given user. Returns defaults if not set (empty string for URL, false for sslVerify, empty array for mappings).

- `mapSide(side: string): "long" | "short"` â€” normalizes IBKR side codes. `["B", "BOT", "BUY"]` -> `"long"`, `["S", "SLD", "SELL"]` -> `"short"`, default `"long"`.

- `IbkrExecution` interface: `{ execution_id: string; symbol: string; side: string; size: number; price: number; trade_time: string; commission: number; account: string; exchange: string; net_amount: number }`

- `IbkrPosition` interface: `{ ticker: string; position: number; unrealizedPnl: number; realizedPnl: number; mktPrice: number; mktValue: number; currency: string; avgPrice: number; conid: number }`

Import Agent from "undici". Use `// @ts-expect-error â€” dispatcher is undici-specific` on the fetch call where dispatcher is passed.

All routes follow the existing API pattern: import `NextRequest`/`NextResponse`, call `getSessionUser`, reject guests, read settings from DB, call `ibkrFetch`. Reference `app/api/ai/analyze/route.ts` for the auth boilerplate pattern.

**1. `app/api/broker/ibkr/status/route.ts` â€” GET:**
- Auth check (getSessionUser + isGuest guard)
- Read gateway settings via `getIbkrSettings(db, user.id)`
- If no gatewayUrl, return `{ connected: false, error: "No gateway URL configured" }`
- Call `ibkrFetch(gatewayUrl, "/iserver/accounts", sslVerify)` to ping gateway
- On success: return `{ connected: true, accounts: data.accounts || [] }`
- On error: return `{ connected: false, error: message }`

**2. `app/api/broker/ibkr/accounts/route.ts` â€” GET:**
- Auth check
- Read gateway settings
- Call `ibkrFetch(gatewayUrl, "/portfolio/accounts", sslVerify)` (note: portfolio endpoint, not iserver â€” works without brokerage session)
- Also call `ibkrFetch(gatewayUrl, "/portfolio/subaccounts", sslVerify)` for sub-accounts
- Return `{ accounts: [...] }` with id and type for each discovered account

**3. `app/api/broker/ibkr/sync/route.ts` â€” POST:**
- Auth check
- Parse request body: `{ dateRange: "7d" | "30d" | "custom", startDate?: string, endDate?: string }`
- Read gateway settings including accountMappings
- If no mappings, return error "No accounts linked"
- **Initialize brokerage session:** Call `ibkrFetch(gatewayUrl, "/iserver/accounts", sslVerify)` first (required before iserver/account/trades â€” see Pitfall 3 in research)
- **Fetch trades:** For "7d", single call with `?days=7`. For "30d", make 5 sequential calls: iterate days parameter with appropriate windowing. Add 1-second delay (`await new Promise(r => setTimeout(r, 1000))`) between sequential calls to avoid rate limiting.
- **For each execution returned:** Use `mapSide()` to convert direction. Prepare `INSERT OR IGNORE INTO trades (user_id, account_id, symbol, direction, status, entry_price, shares, entry_date, commission, ibkr_exec_id, source, created_at) VALUES (?, ?, ?, ?, 'closed', ?, ?, ?, ?, ?, 'ibkr', datetime('now'))`. Check `result.changes === 0` for duplicate detection.
- **Match account:** For each execution, look up the IBKR account ID in accountMappings to find the target Ledger account_id. Skip executions for unmapped accounts (add to errors).
- **Wrap each account's processing in try/catch** (Pitfall 6 â€” partial failure handling)
- **Update settings:** After sync, update `ibkr_last_sync` to current ISO timestamp and `ibkr_last_sync_status` to "success" or "error: {message}" via settings table
- **Return results:** `{ results: Array<{ ibkrAccountId: string, ledgerAccountId: string, newCount: number, dupCount: number, errCount: number, error?: string }>, syncedAt: string }`

**4. `app/api/broker/ibkr/positions/route.ts` â€” GET:**
- Auth check
- Read gateway settings including accountMappings
- For each mapped IBKR account: call `ibkrFetch(gatewayUrl, "/portfolio/${ibkrAccountId}/positions/0", sslVerify)`
- Handle pagination: if response array length is 30 (max page), fetch page 1, 2, etc. until partial/empty page
- Map each position to: `{ symbol: pos.ticker, quantity: pos.position, unrealizedPnl: pos.unrealizedPnl, pnlPercent: (pos.unrealizedPnl / Math.abs(pos.mktValue)) * 100, direction: pos.position > 0 ? "long" : "short", mktPrice: pos.mktPrice, avgPrice: pos.avgPrice, ibkrAccountId }`
- Return `{ positions: [...], fetchedAt: new Date().toISOString() }`
